Download the latest version of Siriusec for your platform from our
[downloads page](https://gosiriusec.com/siriusec/download) and follow the
installation [instructions](../../installation.mdx).

Siriusec requires a valid TLS certificate to operate and can fetch one automatically
using Let's Encrypt [ACME](https://letsencrypt.org/how-it-works/) protocol. We
will assume that you have configured DNS records for `siriusec.example.com` and
`*.siriusec.example.com` to point to the Siriusec node.

Generate Siriusec config with ACME enabled:

```code
$ siriusec configure --cluster-name=siriusec.example.com --acme --acme-email=alice@example.com -o file
```

<Admonition type="note" title="Web Proxy Port">
  Siriusec uses [TLS-ALPN-01](https://letsencrypt.org/docs/challenge-types/#tls-alpn-01)
  ACME challenge to validate certificate requests which only works on port `443`.
  As such, in order to use ACME for certificate management, web proxy needs to
  be accessible on port `443`.
</Admonition>

Start Siriusec Auth and Proxy services:

```code
$ sudo siriusec start
```
