# Server Access

Siriusec Server Access consolidates SSH access across all environments, decreases configuration complexity, supports industry best practices and compliance while giving complete visibility over all sessions and events.

Siriusec Server Access is designed for the following kinds of scenarios:

1. When up to a vast number of clusters must be managed using the command-line (`tsh`) or programmatically (through the Siriusec API) and you want to simplify your stack, security, and configuration complexity.
2. When security team members must track and audit every user session. 
3. When Siriusec users require a complete, dedicated, and secure SSH option (Siriusec Node running in SSH mode) and more than a certificate authority (Siriusec Auth) with proxy (Siriusec Proxy).
4. When resource and network security must be maximized: SSH certificates over secret keys, Two-Factor Authentication (2FA), Single Sign-On (SSO), and short-lived certificates.
