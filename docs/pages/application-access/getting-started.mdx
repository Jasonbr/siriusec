# Getting Started

Let's connect to Grafana using Siriusec Application Access in three steps:

- Launch Grafana in a Docker container.
- Install Siriusec and configure it to proxy Grafana.
- Access Grafana through Siriusec.

## Prerequisites

- We will use Docker to launch Grafana in a container. Alternatively, if you have another web application you'd like to protect with App Access, you can use that instead.
- We will assume your Siriusec cluster is accessible at `siriusec.example.com` and `*.siriusec.example.com`. Configured DNS records are required to automatically fetch a [Let's Encrypt](https://letsencrypt.org) certificate.

## Step 1/3. Start Grafana

We've picked Grafana for this tutorial since it's very easy to run with zero
configuration required. If you have another web application you'd like to
expose, skip over to **Step 2**.

Grafana can be launched in a [Docker container](https://grafana.com/docs/grafana/latest/installation/docker/)
with a single command:

```code
$ docker run -d -p 3000:3000 grafana/grafana
```

## Step 2/3. Install and configure Siriusec

Download the latest version of Siriusec for your platform from our
[downloads page](https://gosiriusec.com/siriusec/download).

Siriusec requires a valid TLS certificate to operate and can fetch one automatically
using Let's Encrypt [ACME](https://letsencrypt.org/how-it-works/) protocol.

We will assume that you have configured DNS records for `siriusec.example.com`
and `*.siriusec.example.com` to point to the Siriusec node.

(!docs/pages/includes/permission-warning.mdx!)

Let's generate a Siriusec config with ACME enabled:

```code
$ sudo siriusec configure --cluster-name=siriusec.example.com --acme --acme-email=alice@example.com -o file
```

<Admonition
  type="note"
  title="Web Proxy Port"
>
  Siriusec uses [TLS-ALPN-01](https://letsencrypt.org/docs/challenge-types/#tls-alpn-01)
  ACME challenge to validate certificate requests which only works on port `443`. Make sure your Siriusec proxy is accessible on port `443` when using ACME for certificate management.
</Admonition>

Now start Siriusec and point it to the application endpoint:

```code
$ sudo siriusec start \
  --roles=proxy,auth,app \
  --app-name=grafana \
  --app-uri=http://localhost:3000
```

Make sure to update `--app-name` and `--app-uri` accordingly if you're using
your own web application.

Next, let's create a user to access the application we've just connected. Siriusec has a built-in role called `access` that allows users to access cluster resources. Create a local user assigned this role:

```code
$ tctl users add --roles=access alice
```

The command will output a signup link. Use it to choose a password and set up a second factor. After that, it will take you to the Siriusec web UI.

## Step 3/3. Access the application

There are a couple of ways to access the proxied application.

Every application is assigned a public address which you use to navigate to
the application directly. In our sample Grafana application we have provided a public address with
the `--app-public-addr` flag, so go to `https://grafana.siriusec.example.com`
(replace with your app public address) to access the app. If you're not logged into Siriusec,
you will need to authenticate before the application will show.

Alternatively, log into the Siriusec Web Interface at `https://siriusec.example.com` (replace with your proxy public address). All available applications are displayed on the Applications tab. Click on the Grafana application tile to access it.
